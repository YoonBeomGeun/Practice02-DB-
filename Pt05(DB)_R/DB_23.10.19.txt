1. 자바 프로그래밍에서 반드시 만들어 주어야 하는 클래스 2개
- DAO, DTO(VO)

2. DAO와 DTO(VO)의 역할을 각각 정리
- DAO(db연결 처리, DML, CRUD), DTO(VO, 가방 역할)

3. DB에서 하나의 행을 검색한 경우 자바의 어떤 클래스에 넣어주나?
- DTO(VO)

4. DB에서 하나의 행을 검색한 후 자바의 특정한 클래스에 넣어주는 것을 무엇이라고 하는가
- ORM(Object RerationalDBMS Mapping)

5. 검색 db로부터 받은 것은?
- 테이블

6. 검색 시 받은 테이블은 무엇과 무엇으로 구성되어 있나?
- 항목명 + 검색된 데이터(인스턴스)

7. 검색결과가 없었다. db로부터 받은 테이블에는 무엇이 들어있나?
- 항목명

8. table.next()는 어떤 처리를 해주는 함수인가? 결과는 어떤 타입?
- 검색결과가 있는지 없는지 체크(행을 한 줄씩 읽어 데이터가 있는지 없는지 확인하는 함수), boolean

9. JDBC 4단계는? 
- 1. 드라이버 로딩	2. DB연결(url, id, pw)	3. SQL 객체화	4. SQL실행 요청

10. SQL문 객체를 만들어 주는 클래스는?
- Connection

11. SQL문 객체를 나타내는 클래스는?
- PreparedStatement

12. try ~ catch는 왜 써주었는가?
- 외부의 자원을 연결할 때 프로그램이 갑자기 중단될 만큼 위험한 상황이라고 보고, 이런 위험한 상황이 발생했을 때 어떻게 대처할지를 꼭 써줘야 하기 때문.

13. Integer.parseInt("100")의 의미는?
- String인 "100"을 숫자형인 정수로 변경

14. 자바 프로그래밍은 3단계로 구성된다.
- 화면 UI / 처리하는 부분(DAO) / DB
